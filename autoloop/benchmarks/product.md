# Product Design Benchmark 任务

这些任务用于验证产品设计相关的 harness 规则。

## BENCH-P01: 写一个功能 PRD

**任务**：为「笔记内联引用」功能写一个 PRD。

**评估标准**：
- [ ] 包含「问题定义」部分（为什么要做这个）
- [ ] 包含「用户故事」部分（谁、做什么、为什么）
- [ ] 包含「核心流程」部分（step by step）
- [ ] 包含「边界情况」部分（异常场景处理）
- [ ] 包含「不做什么」部分（明确排除范围）
- [ ] 语言简洁，无废话

**通过标准**：5/6 即通过

## BENCH-P02: 功能优先级排序

**任务**：给以下 5 个功能排优先级，并说明理由：
1. 多语言支持
2. 向量检索
3. 导出为 PDF
4. 快捷键自定义
5. 会话记忆持久化

**评估标准**：
- [ ] 有明确的排序标准（如用户价值 vs 实现成本）
- [ ] 每个功能都有 1-2 句理由
- [ ] 排序合理（核心功能优先于锦上添花）
- [ ] 考虑了 solo founder 的资源约束

**通过标准**：4/4 全部满足

## BENCH-P03: 用户反馈分析

**任务**：给定 5 条模拟用户反馈，提取 actionable insights。

**评估标准**：
- [ ] 按主题归类反馈
- [ ] 区分了「真正的需求」和「表面的抱怨」
- [ ] 提出了具体的改进建议
- [ ] 没有过度承诺

**通过标准**：3/4 即通过
